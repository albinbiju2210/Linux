Understanding Linux Commands

Linux commands are text-based instructions used to interact with the operating system. They provide powerful tools for file manipulation, system administration, and process management. The commands listed below focus on essential file and directory operations that every Linux user should know. Each command comes with various options (flags) that modify its behavior to suit specific needs.

Pro Tip: Most commands support the --help flag or man command for detailed documentation.

ls

List directory contents

ls [options] [directory]

Purpose

The ls command is used to list files and directories in Linux. It's one of the most frequently used commands that helps you:

  • View directory contents
  • Check file permissions and ownership
  • See file sizes and modification times
  • Identify file types through colors

Common Usage

  • ls -l: Long listing format

    Shows detailed information including permissions, owner, size, and modification time

  • ls -a: Show hidden files

    Displays all files, including those starting with a dot (.)

  • ls -h: Human readable sizes
  • ls -R: Recursive listing

Practical Examples

1. List all files with details:

ls -lah 
total 128K
drwxr-xr-x  2 user user 4.0K Sep 15 14:30 .
drwxr-xr-x 15 user user 4.0K Sep 15 14:28 ..
-rw-r--r--  1 user user  12K Sep 15 14:30 document.pdf
-rw-r--r--  1 user user 8.2K Sep 15 14:29 image.jpg

2. Sort files by size:

ls -lS

Lists files in descending order of size

Important Notes

  • Color coding helps identify file types:
    • Blue: Directories
    • Green: Executable files
    • Red: Compressed archives
    • Cyan: Symbolic links
  • Use with grep to filter results: ls -l | grep ".txt"
  • Combine with other commands using pipes

cd

Change Directory

cd [directory]

Purpose

The cd (Change Directory) command is used to navigate the file system. It helps you:

  • Move between directories
  • Access different locations quickly
  • Navigate relative and absolute paths
  • Return to previous locations

Common Usage

  • cd ..: Go up one directory 
    $ pwd
/home/user/documents
$ cd ..
$ pwd
/home/user

    Moves to the parent directory

  • cd ~: Go to home directory 
    $ cd ~
$ pwd
/home/user

    Returns to user's home directory

Practical Examples

1. Navigate to specific directory:

cd /var/www/html 
$ cd /var/www/html
$ pwd
/var/www/html

2. Return to previous directory:

cd - 
$ pwd
/var/www/html
$ cd -
/home/user

Important Notes

  • Directory shortcuts:
    • . - Current directory
    • .. - Parent directory
    • ~ - Home directory
    • - - Previous directory
  • Best practices:
    • Use Tab for auto-completion
    • Use absolute paths for scripts
    • Check pwd after changing directories
    • Use relative paths for nearby locations

pwd

Print Working Directory

pwd [options]

Purpose

The pwd (Print Working Directory) command displays your current location in thefilesystem. It helps you:

  • Verify your current directory location
  • Get absolute path for scripts
  • Debug path-related issues
  • Navigate complex directory structures

Common Usage

  • pwd -P: Show physical path 
    $ ln -s /var/www/html webroot
$ cd webroot
$ pwd -P
/var/www/html

    Shows actual path instead of symbolic link path

  • pwd -L: Show logical path 
    $ pwd -L
/home/user/webroot

    Shows path with symbolic links preserved

Practical Examples

1. Use in shell scripts:

CURRENT_DIR=$(pwd) 
#!/bin/bash
CURRENT_DIR=$(pwd)
echo "Working from: $CURRENT_DIR"
cd /tmp
# Do some work
cd "$CURRENT_DIR"  # Return to original directory

2. Combine with other commands:

echo "Current location: $(pwd)" 
$ echo "Current location: $(pwd)"
Current location: /home/user/projects

Important Notes

  • Path information:
    • Always shows absolute path from root (/)
    • Built into shell for efficiency
    • Essential for shell scripts
    • Handles special characters in paths
  • Best practices:
    • Quote the output when using in scripts
    • Use -P when working with symlinks
    • Combine with cd to track directory changes
    • Store path in variable for later use

mkdir

Make Directory

mkdir [options] directory

Purpose

The mkdir command creates new directories in Linux. It helps you:

  • Create single or multiple directories
  • Set up directory structures
  • Create parent directories automatically
  • Set permissions during creation

Common Usage

  • mkdir -p: Create parent directories 
    $ mkdir -p projects/website/css
$ tree projects/
projects/
└── website/
    └── css/

    Creates all necessary parent directories

  • mkdir -m: Set directory permissions 
    $ mkdir -m 755 public_files
$ ls -ld public_files
drwxr-xr-x 2 user user 4096 Mar 15 10:30 public_files

    Creates directory with specific permissions

Practical Examples

1. Create multiple directories:

mkdir docs images videos 
$ mkdir docs images videos
$ ls -l
drwxr-xr-x 2 user user 4096 Mar 15 10:31 docs
drwxr-xr-x 2 user user 4096 Mar 15 10:31 images
drwxr-xr-x 2 user user 4096 Mar 15 10:31 videos

2. Create project structure:

mkdir -p project/{src,docs,tests} 
$ mkdir -p project/{src,docs,tests}
$ tree project/
project/
├── src/
├── docs/
└── tests/

Important Notes

  • Directory creation:
    • Default permissions are modified by umask
    • Parent directories must exist or use -p
    • Directory names are case-sensitive
    • Cannot create directories with same name
  • Best practices:
    • Use meaningful directory names
    • Avoid spaces in directory names
    • Consider permissions carefully
    • Use -p when creating nested structures

rm/rmdir

Remove files or directories

rm [options] file rmdir [options] directory

Purpose

The rm command removes files and directories permanently. It helps you:

  • Delete unwanted files
  • Remove empty directories
  • Clean up filesystem space
  • Delete directory trees

Common Usage

  • rm -r: Remove recursively 
    $ rm -r old_project/
$ ls -l
total 0

    Removes directory and all its contents

  • rm -i: Interactive mode 
    $ rm -i important.txt
rm: remove regular file 'important.txt'? y

    Asks for confirmation before deletion

Practical Examples

1. Remove multiple files:

rm file1.txt file2.txt file3.txt 
$ ls
file1.txt file2.txt file3.txt
$ rm file1.txt file2.txt file3.txt
$ ls

2. Remove empty directory:

rmdir empty_dir/ 
$ mkdir test_dir
$ rmdir test_dir
$ ls -l
total 0

Important Notes

  • Safety considerations:
    • ⚠️ Deletion is permanent - no recycle bin
    • Use -i flag for interactive deletion
    • Be extremely careful with rm -rf
    • Double-check wildcards before executing
  • Best practices:
    • Create backups before mass deletions
    • Use rm -i when learning
    • Consider using trash-cli instead
    • Never run rm -rf / or rm -rf /*

cp

Copy files and directories

cp [options] source destination

Purpose

The cp (copy) command creates copies of files and directories. It helps you:

  • Create file backups
  • Duplicate directory structures
  • Share files between locations
  • Preserve file attributes

Common Usage

  • cp -r: Copy recursively 
    $ cp -r project/ backup/
$ tree backup/
backup/
└── project/
    ├── src/
    ├── docs/
    └── README.md

    Copies entire directory structure

  • cp -p: Preserve attributes 
    $ cp -p script.sh backup/
$ ls -l script.sh backup/script.sh
-rwxr-xr-x 1 user user 1234 Mar 15 10:30 script.sh
-rwxr-xr-x 1 user user 1234 Mar 15 10:30 backup/script.sh

    Maintains timestamps, permissions, and ownership

Practical Examples

1. Create backup with timestamp:

cp config.ini config.ini.$(date +%Y%m%d) 
$ cp config.ini config.ini.20240315
$ ls -l config.ini*
-rw-r--r-- 1 user user 2048 Mar 15 10:30 config.ini
-rw-r--r-- 1 user user 2048 Mar 15 10:30 config.ini.20240315

2. Copy multiple files:

cp -v *.txt backup/ 
$ cp -v *.txt backup/
'file1.txt' -> 'backup/file1.txt'
'file2.txt' -> 'backup/file2.txt'
'file3.txt' -> 'backup/file3.txt'

Important Notes

  • Safety considerations:
    • Use -i to prevent overwriting files
    • Check destination space availability
    • Verify copy success with diff
    • Be careful with symbolic links
  • Best practices:
    • Use -p to preserve attributes
    • Add -v for verbose output
    • Consider using rsync for large copies
    • Create dated backups of important files

mv

Move or rename files

mv [options] source destination

Purpose

The mv (move) command moves or renames files and directories. It helps you:

  • Rename files and directories
  • Move files between locations
  • Organize filesystem structure
  • Backup files with new names

Common Usage

  • mv -i: Interactive mode 
    $ mv -i important.txt ~/backup/
mv: overwrite '/home/user/backup/important.txt'? y

    Asks before overwriting existing files

  • mv -v: Verbose output 
    $ mv -v *.txt docs/
'file1.txt' -> 'docs/file1.txt'
'file2.txt' -> 'docs/file2.txt'

    Shows what is being moved

Practical Examples

1. Rename a file:

mv oldname.txt newname.txt 
$ ls
oldname.txt
$ mv oldname.txt newname.txt
$ ls
newname.txt

2. Move multiple files:

mv file1.txt file2.txt directory/ 
$ ls
file1.txt file2.txt directory/
$ mv file1.txt file2.txt directory/
$ ls directory/
file1.txt file2.txt

Important Notes

  • Safety considerations:
    • Use -i to prevent accidental overwrites
    • Check destination space availability
    • Verify source and destination paths
    • Be careful with wildcards
  • Best practices:
    • Use -v for visibility of operations
    • Keep meaningful file names
    • Check permissions before moving
    • Backup important files first

cat

Concatenate and display files

cat [options] [file...]

Purpose

The cat (concatenate) command displays and combines file contents. It helps you:

  • View file contents quickly
  • Combine multiple files
  • Create new files from input
  • Display line numbers

Common Usage

  • cat -n: Number all lines 
    $ cat -n file.txt
     1	First line
     2	Second line
     3	Third line

    Shows line numbers for all lines

  • cat -b: Number non-blank lines 
    $ cat -b file.txt
     1	First line

     2	Third line

    Numbers only non-empty lines

Practical Examples

1. Combine multiple files:

cat file1.txt file2.txt > combined.txt 
$ cat file1.txt
Hello
$ cat file2.txt
World
$ cat file1.txt file2.txt > combined.txt
$ cat combined.txt
Hello
World

2. Create a new file:

cat > newfile.txt 
$ cat > newfile.txt
Type your text here
Press Ctrl+D when done
$ cat newfile.txt
Type your text here
Press Ctrl+D when done

Important Notes

  • Common uses:
    • Quick file viewing
    • File concatenation
    • Creating small text files
    • Displaying file contents with numbers
  • Best practices:
    • Use less for large files
    • Redirect output for large combinations
    • Check file type before viewing
    • Use -A to see hidden characters

touch

Create empty files or update timestamps

touch [options] file

Purpose

The touch command creates empty files and updates timestamps. It helps you:

  • Create new empty files
  • Update file access times
  • Update file modification times
  • Synchronize timestamps between files

Common Usage

  • touch -a: Update access time only 
    $ ls -lu file.txt
-rw-r--r-- 1 user user 0 Mar 14 10:00 file.txt
$ touch -a file.txt
$ ls -lu file.txt
-rw-r--r-- 1 user user 0 Mar 15 10:30 file.txt

    Only the access time is updated

  • touch -r: Use reference file 
    $ touch -r reference.txt target.txt
$ ls -l reference.txt target.txt
-rw-r--r-- 1 user user 0 Mar 15 10:30 reference.txt
-rw-r--r-- 1 user user 0 Mar 15 10:30 target.txt

    Copies timestamps from reference file

Practical Examples

1. Create multiple files:

touch file{1..3}.txt 
$ touch file{1..3}.txt
$ ls -l
-rw-r--r-- 1 user user 0 Mar 15 10:30 file1.txt
-rw-r--r-- 1 user user 0 Mar 15 10:30 file2.txt
-rw-r--r-- 1 user user 0 Mar 15 10:30 file3.txt

2. Set specific timestamp:

touch -t 202403151030.00 file.txt 
$ touch -t 202403151030.00 file.txt
$ ls -l file.txt
-rw-r--r-- 1 user user 0 Mar 15 10:30 file.txt

Important Notes

  • File creation:
    • Creates empty files if they don't exist
    • Updates timestamps if files exist
    • Doesn't modify file content
    • Requires write permission in directory
  • Best practices:
    • Use descriptive file names
    • Check file existence first
    • Verify timestamp changes
    • Consider file permissions

grep

Search text patterns in files

grep [options] pattern [file...]

Purpose

The grep command searches for text patterns in files. It helps you:

  • Find specific text in files
  • Filter command output
  • Search through multiple files
  • Count pattern occurrences

Common Usage

  • grep -i: Case insensitive search 
    $ grep -i "error" log.txt
Error: Connection failed
WARNING: error in line 42
System ERROR detected

    Matches text regardless of case

  • grep -r: Recursive search 
    $ grep -r "TODO" ./src/
./src/main.c:// TODO: Implement error handling
./src/utils.h:/* TODO: Add documentation */

    Searches in all files under directory

Practical Examples

1. Search with context:

grep -C 2 "error" log.txt 
System starting
Initializing...
Error: Connection failed
Retrying connection...
Connection established

2. Count matches:

grep -c "error" *.log 
$ grep -c "error" *.log
app.log:5
system.log:3
debug.log:8

Important Notes

  • Search features:
    • Supports regular expressions
    • Can highlight matches in color
    • Works with pipes and redirections
    • Handles multiple file types
  • Best practices:
    • Use -i for case-insensitive search
    • Add context with -A, -B, or -C
    • Quote patterns with spaces
    • Use -w for whole word matches
